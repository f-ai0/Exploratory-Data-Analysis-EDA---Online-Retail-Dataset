 Exploratory Data Analysis (EDA) - Online Retail Dataset

Dataset Source
The dataset used in this project is the Online Retail Dataset, available from:
https://www.kaggle.com/datasets

Dataset Overview
- Number of Rows: ~540,000
- Number of Columns: 8

Features Description
- InvoiceNo: Unique identifier for each transaction
- StockCode: Unique product code
- Description: Name of the product
- Quantity: Number of products purchased
- InvoiceDate: Date and time of the transaction
- UnitPrice: Price per unit of the product
- CustomerID: Unique identifier for each customer
- Country: Country of the customer

Purpose of the Dataset
The purpose of this dataset is to analyze retail sales data and understand customer purchasing behavior. This includes identifying sales trends, top products, and patterns across different countries.

Data Cleaning
The following data cleaning steps were performed:
- Removed missing values (especially CustomerID)
- Removed duplicate rows
- Converted InvoiceDate to datetime format
- Ensured correct data types for analysis

These steps were necessary to improve data quality and ensure accurate analysis.

 Data Analysis & Visualization
Several visualizations were created, including:
- Distribution of Quantity
- Sales by Country
- Sales excluding United Kingdom
- Monthly Sales Trends
- Top 10 Products by Quantity
- Correlation between Quantity and Unit Price

These visualizations help identify trends and patterns in the data.

 Key Insights
- The United Kingdom dominates the sales data compared to other countries.
- Some products are significantly more popular than others.
- Sales vary across months, showing clear trends.
- There is a weak relationship between quantity and unit price.

 Conclusion
This analysis provides useful insights into retail sales patterns, helping to understand customer behavior and product demand.
